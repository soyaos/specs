# SoyaOS CLI v0 — Canonical Surface

This is the contract the `soyaos` binary implements. CLI changes happen here first, then in code.

## Spec version

```
soyaos --spec-version
# → cli.v0
```

## Top-level commands

```
soyaos start         [--listen 127.0.0.1:7474] [--rpc 127.0.0.1:7475] [--data-dir $XDG_DATA_HOME/soyaos]
soyaos version
soyaos help
soyaos agent <subcommand>
soyaos channel <subcommand>     # planned for Stage 2-3
soyaos schedule <subcommand>    # planned for Stage 2-3
```

## `soyaos start`

Boots the Solo edition all-in-one process (Planet + Moon + Comet in-process).

Flags:

- `--listen` — HTTP listen address for the OpenAI-Compat gateway, `/v1/*` and Studio. **Default: `127.0.0.1:7474`.**
- `--rpc` — local control-plane RPC for `agent` / `channel` / `schedule` subcommands. **Default: `127.0.0.1:7475`.**
- `--data-dir` — on-disk persistence root (Bolt / SQLite / artifacts). **Default: `$XDG_DATA_HOME/soyaos` → fallback `~/.local/share/soyaos`.**

Environment:

- `SOYA_MODEL_API_KEY` — if set, used as the dev API key instead of the auto-seeded `sk-soya-dev-local`.

Exit codes:

- `0` — clean shutdown (SIGINT / SIGTERM).
- `1` — runtime error.
- `2` — bad usage.

Health probe:

```
GET /healthz                → 200, body `ok` (plain text)
GET /healthz?format=json    → 200, body {"status":"ok","edition":"solo","version":"<semver>","agents":N}
```

## `soyaos version`

Prints the build identification block. Also accepts `-v` and `--version`.

## `soyaos agent`

```
soyaos agent create <name>           Scaffold a new Agent in ./<name>/ from the SoyaPack v0 Agent template.
soyaos agent list                    List Agents registered with the running soyaos (via :7475).
soyaos agent run <slug> "<prompt>"   Invoke an Agent once, print the response.
soyaos agent build [<path>]          Build a SoyaPack from a directory (default: cwd). Writes <name>-<version>.spk.
soyaos agent deploy <pack.spk>       Register a built pack with the running soyaos.
```

All `agent` subcommands except `create` and `build` require a running `soyaos start`.

## `soyaos channel` (planned)

```
soyaos channel bind <kind> <name>    OAuth / webhook setup wizard for an inbound/outbound channel.
soyaos channel list                  List bound channels.
soyaos channel test <name>           Send a probe message to verify connectivity.
```

## `soyaos schedule` (planned)

```
soyaos schedule add <agent> <cron-or-once>   Register a cron / one-shot trigger for an Agent.
soyaos schedule list
soyaos schedule cancel <id>
```

## Output format conventions

- Default output is human-readable text.
- `--json` (where available) emits a single JSON value to stdout; structured logs go to stderr.
- All commands accept `--quiet` to suppress non-essential stderr output.
- Errors always include a `code` field in the JSON form for programmatic handling.

## What's NOT in v0

- `soyaos cluster join` / Planet-Moon-Comet split-mode flags — Cluster edition only.
- Plugin / Market commands.
- Studio launch flag — Studio is served from `--listen` directly.
