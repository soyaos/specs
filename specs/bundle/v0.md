# SoyaPack Bundle Layout v0

A SoyaPack **bundle** is the on-the-wire form a Pack travels in: it can be (a) a directory on disk, (b) a single `.tar.zst` archive, or (c) an OCI artifact pushed to a registry. All three forms carry the same canonical contents.

## OCI mediaType

```
application/vnd.soyaos.pack.v1+json
```

(The `v1` lives in the mediaType because mediaTypes are how OCI consumers discriminate format families. The *content* under that mediaType is still pre-stable, hence spec `v0`.)

## Archive form (`.tar.zst`)

Filename convention: `<name>-<version>.spk` (Soya Pack). Internal layout:

```
<name>-<version>.spk        # tar+zstd
├── soyapack.yaml           # manifest, REQUIRED
├── capabilities.yml        # OPTIONAL — may be inlined in manifest
├── sbom.spdx.json          # SPDX 2.3
├── signature.cose          # COSE_Sign1 over canonical(manifest) ++ contents.merkle
├── manifest.canonical.json # the bytes that were actually signed (parsers reject mismatches)
├── runtime-hints.yml       # OPTIONAL
├── prompts/                # Agent prompts
├── templates/              # output templates
├── tools/                  # tool descriptors (MCP-style JSON)
├── contents/               # data files (Memory kind)
└── examples/               # input/expected pairs
```

All non-binary files are UTF-8, LF line endings, NFC-normalized.

## Content addressing

The archive's canonical hash is `sha256(<sorted tar entries with stripped mtime>)`. Implementations:

1. Stable tar order: lexicographic by entry name.
2. mtime fixed to `1970-01-01T00:00:00Z` for every entry.
3. uid/gid fixed to 0.
4. Compression *after* tarring; zstd level 19, `--long=27`.

The result is byte-stable across machines.

## Signature

`signature.cose` is a COSE_Sign1 envelope over `manifest.canonical.json ++ contents.merkle`, where:

- `manifest.canonical.json` is the canonical JSON form of `soyapack.yaml`.
- `contents.merkle` is the merkle root of `sha256(name || data)` for every non-signature file in the archive, sorted by name.

Signing key chain: keyless Sigstore (cosign) is the default; HSM-backed PGP is allowed for enterprise Packs.

## Validation steps

A conformant verifier MUST:

1. Untar without honoring uid/gid/mtime (trust nothing inside).
2. Recompute `contents.merkle` and compare against the signed value.
3. Recompute canonical JSON of `soyapack.yaml` and compare bytes to `manifest.canonical.json`. **Mismatch = invalid Pack.**
4. Verify COSE_Sign1 with the configured trust roots.
5. Reject any unknown top-level field in `soyapack.yaml`.

## Registry storage

When pushed to an OCI registry, a Pack is stored as:

- One OCI artifact with `mediaType: application/vnd.soyaos.pack.v1+json`.
- The single layer blob is the `.spk` archive (`mediaType: application/vnd.soyaos.pack.layer.v1.tar+zstd`).
- The manifest's `annotations` MUST include:
  - `ai.soyaos.pack.name`
  - `ai.soyaos.pack.version`
  - `ai.soyaos.pack.kind`
  - `ai.soyaos.pack.spec` = `"soyapack.v0"`

## File extension

The single-archive form is `.spk`. The MIME type when served over HTTP is `application/vnd.soyaos.pack.v1+json`. Browsers should treat it as a download (not preview).
